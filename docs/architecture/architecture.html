Why two serving APIs

recsys-api answers "what to show" (embedding retrieval + learned reranking); agent-api answers "what did you ask for" (tool-calling LLM over the same catalog, plus policy RAG). Same feature/vector store underneath, different query pattern on top.

Training split: local vs. Kaggle

Negative sampling is a CPU-bound O(n×m) loop — it runs locally, free. Two-Tower / SASRec / LTR need GPU throughput, so the prebuilt train/valid/test tensors ship to a Kaggle GPU notebook instead of wasting quota on data prep.

LLM backend is swappable

agent-api targets whichever backend is configured — Ollama for a free CPU baseline, vLLM on GPU Spot for throughput, or Claude as a managed fallback — without touching the graph or tool logic.
